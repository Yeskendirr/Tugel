# Жоба құрылымы және ережелер

Бұл құжат жобаның файлдық құрылымын және код жазу ережелерін сипаттайды.
Барлық қатысушылар осы ережелерді сақтауы тиіс.

---

## 1. Жалпы құрылым

```
Tugel/
├── backend/      # Node.js + Express + TypeScript API
├── frontend/     # React + TypeScript (Vite)
└── database/     # SQL схема
```

Бұл құжат негізінен **frontend** құрылымына арналған.

---

## 2. Frontend құрылымы

```
frontend/src/
├── main.tsx              # кіру нүктесі (index.css осында қосылады)
├── App.tsx               # маршрутизация (routes)
├── index.css             # ГЛОБАЛДЫ стильдер мен айнымалылар
│
├── components/           # ГЛОБАЛДЫ компоненттер (бүкіл сайтта қолданылады)
│   ├── Header/
│   │   ├── Header.tsx
│   │   └── Header.css
│   ├── Footer/
│   │   ├── Footer.tsx
│   │   └── Footer.css
│   └── Pagination/
│       ├── Pagination.tsx
│       └── Pagination.css
│
├── pages/                # БЕТТЕР
│   ├── admin/            # әкімші беттері
│   │   └── ...
│   └── user/             # қолданушы беттері
│       └── ...
│
├── context/              # React Context (мысалы, AuthContext)
├── services/             # API сұраулары (axios)
├── types/                # TypeScript типтері
└── assets/               # суреттер, шрифттер
```

---

## 3. Бет (page) құрылымы — НЕГІЗГІ ЕРЕЖЕ

Әр беттің **жеке папкасы** болады. Папка ішінде:

```
pages/user/Home/
├── Home.tsx              # беттің негізгі компоненті
├── Home.css              # беттің стильдері
└── components/           # ТЕК осы бетке қатысты компоненттер
    ├── HeroSection/
    │   ├── HeroSection.tsx
    │   └── HeroSection.css
    └── StatsCard/
        ├── StatsCard.tsx
        └── StatsCard.css
```

**Мақсаты:** бетті кіші компоненттерге бөлу — код таза әрі қайта қолдануға ыңғайлы болады.

### Компонент қайда орналасады?

- Компонент **бір бетте ғана** қолданылса → сол беттің `components/` папкасында.
- Компонент **бірнеше бетте** қолданылса → глобалды `src/components/` папкасында.

---

## 4. Атау ережелері

- Папка мен компонент аты — **PascalCase**: `Home`, `EquipmentTable`, `Header`.
- Әр компонентке өз `.css` файлы: `Home.tsx` → `Home.css`.
- Компонент файлы мен папка аты бірдей болуы керек.

---

## 5. Стиль (CSS) ережелері

- Глобалды айнымалылар (түстер, аралықтар, шрифт) **тек `index.css`** ішінде.
- Әр компонент өз `.css` файлында тек өзінің стилін жазады.
- Түс, аралық, радиус — `index.css`-тегі `var(--...)` арқылы алынады.

### Дизайн талаптары (МІНДЕТТІ)

- **Минимализм.** Артық әшекейсіз, таза әрі әдемі.
- **Градиент ҚОЛДАНУҒА БОЛМАЙДЫ.** Тек біркелкі түстер.
- **Эмодзи ҚОЛДАНУҒА БОЛМАЙДЫ** (интерфейсте де, кодта да, коммиттерде де).
- **Иконка кітапханалары ҚОЛДАНУҒА БОЛМАЙДЫ** (lucide-react, react-icons т.б.).
  Қажет болса — қарапайым SVG немесе мәтін.
- Түс палитрасы шектеулі, контраст жеткілікті болсын.

---

## 6. Тіл

- Интерфейс **толығымен қазақ тілінде**.
- Код ішіндегі түсініктемелер (комментарийлер) — қазақ тілінде.

---

## 7. Қысқаша қорытынды

1. Әр бет = жеке папка (`Бет.tsx` + `Бет.css` + керек болса `components/`).
2. Глобалды компоненттер → `src/components/`.
3. Беттер → `pages/admin/` немесе `pages/user/`.
4. Глобалды стильдер тек `index.css`-те.
5. Минимализм, градиентсіз, эмодзисіз, иконка кітапханаларынсыз.
